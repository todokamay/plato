VideoAutoPipeline Tools

Plato Quality Gate

Advanced

1. Detect Outputs

Find real VideoAutoPipeline MP4 output folders and prefill the input folder when possible.

2. Select Input Folder

If no folder is found, create data\temp\manual_videoautopipeline_outputs, put one or more .mp4 files inside, then use that path here.

3. Dry Run

Run one safe scan. This does not call ffmpeg and does not modify originals.

4. One-Shot Auto QC

Run the existing conservative Auto QC pipeline once for the selected folder.

5. Safe Replace Mode

Default mode is copy-only. Replace mode creates backups first. Only accepted fixed clips replace originals. Rollback uses replace_log.json.

6. Batch QC Once

Analyze and route a folder once without starting continuous watch mode.

7. Continuous Watch Mode

Start or stop one background watcher for the selected folder.

8. VideoAutoPipeline → Plato Flow

VideoAutoPipeline runs first. Telegram happens only after Plato approval. Dry-run is default. No Telegram send unless explicitly checked and env vars exist.

Cleanup danger zone

9. VideoAutoPipeline Delivery Gate

Telegram only sends Plato-approved outputs. Missing Telegram env means send is skipped.

10. Production Diagnostics

Export a sanitized diagnostics bundle for debugging. No video files or SQLite databases are included.

py tools\export_production_diagnostics.py
